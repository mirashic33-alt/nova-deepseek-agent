# process_daily_digest — обработка дневных логов в вики-статьи

## Когда вызывать
При получении пинка (свободный полёт, no specific topic from user). Проверить `workspace/memory/` — есть ли новые записи со времени последней обработки.

## Как обрабатывать
1. Прочитать `workspace/memory/ГГГГ-ММ-ДД.md` за сегодня (и вчера, если вчерашний не обработан)
2. Найти в логах значимые события:
   - code changes
   - new project facts
   - важные договорённости и решения
   - новые люди/агенты
   - баги и фиксы
3. Для каждого значимого события:
   - если есть подходящая статья в вики — добавить в неё новый раздел (append_file)
   - если темы нет — создать новую статью в `workspace/вики/`
   - обновить ссылки `[[...]]` где нужно
4. Обновить `workspace/вики/INDEX.md` — добавить новые статьи в оглавление
5. Записать маркер `.вики_обработано` с датой последней обработки

## Что НЕ писать в вики
- Шум: разговоры «сколько времени», приколы без последствий
- Повторы: если то же самое уже есть в статье
- Черновики и незаконченные обсуждения

## Формат статей
- Заголовок: `# Название статьи`
- Разделы: `## Подраздел`
- Перекрёстные ссылки: `[[Имя_статьи]]`
- Ссылки на источники внизу: `См. [[...]] · [[...]]`
